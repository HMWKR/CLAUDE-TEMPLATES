## 12. 에러 핸들링 + 환각 방지

### 12.1 에러 핸들링 매트릭스

| 실패 시나리오 | 감지 방법 | Fallback 전략 |
|:------------:|----------|:------------:|
| TM 개별 실패 | 리포트 파일 미생성 | Lead가 해당 TM 롤플레이 |
| Wave 2 전체 실패 | 6개 리포트 모두 미생성 | Wave 1 결과만으로 basic 리포트 |
| Wave 3 전체 실패 | 6개 리포트 모두 미생성 | Wave 1+2 결과로 --pro 리포트 |
| agent-teams 비활성 | 환경변수 미설정 | 단일 에이전트 순차 실행 (T1만) |
| Playwright MCP 실패 | 브라우저 연결 에러 | 코드 기반 정적 분석 대체 |
| CSS evaluate 실패 | JS 실행 에러 | 해당 토큰 스킵 + [NOT-TESTABLE] |
| 데이터 디렉토리 생성 실패 | 파일시스템 에러 | 메모리 기반 데이터 전달 |

### 12.2 agent-teams 비활성 시 단일 에이전트 Fallback

```
[Fallback 모드 - 단일 에이전트]
IF CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS ≠ 1:
  1. Lead 단독으로 Stage 0 + Stage 1 수행
  2. Wave 1의 T1 항목만 순차 검사 (~54항목)
  3. 15차원 UX Score 산출 (검사된 차원만)
  4. 미검사 차원은 [NOT-TESTED] 표시
  5. basic 수준 리포트 생성
```

### 12.3 환각 방지 프로토콜 (6가지 검증 마커)

모든 TM 리포트에서 각 발견사항은 반드시 검증 마커를 포함해야 한다:

| 마커 | 의미 | 사용 조건 |
|------|------|----------|
| `[DATA-VERIFIED]` | CSS/HTML 데이터 파일에서 직접 확인 | 토큰 JSON 데이터 기반 |
| `[SNAPSHOT-VERIFIED]` | browser_snapshot에서 확인 | 스냅샷 텍스트 기반 |
| `[PATTERN-INFERRED]` | 패턴 기반 추론 (80%+ 확신) | 데이터 불완전 시 |
| `[CROSS-REFERENCED]` | 다른 Wave/TM 리포트에서 교차 확인 | Wave 2+3 교차 참조 |
| `[FRAMEWORK-BASED]` | 전문 프레임워크 기준 적용 | 이론적 판단 |
| `[NOT-TESTABLE]` | 현재 수집 데이터로 검증 불가 | 자동 점수 제외 |

#### 마커 사용 규칙

```
필수: 모든 발견사항에 최소 1개 마커
금지: 마커 없는 발견사항 보고 금지
우선순위: DATA-VERIFIED > SNAPSHOT-VERIFIED > CROSS-REFERENCED > PATTERN-INFERRED > FRAMEWORK-BASED
신뢰도:
  - [DATA-VERIFIED] + [SNAPSHOT-VERIFIED] = 최고 신뢰 (이중 확인)
  - [PATTERN-INFERRED] 단독 = 최소 신뢰 (추가 검증 권장 태그)
  - [NOT-TESTABLE] = 점수 산출 제외, 리포트에는 포함
```

### 12.4 TM 리포트 필수 구조

각 Teammate의 리포트는 다음 구조를 반드시 따른다:

```markdown
# {카테고리명} 분석 리포트 — {TM 번호}

## 분석 범위
- 카테고리: {카테고리 ID + 이름}
- 검사 항목: {N}개 (Tier {적용 범위})
- 참조 데이터: {읽은 파일 목록}
- 교차 참조: {참조한 Wave 리포트} (Wave 2+ 해당)

## 발견사항

### ✅ 통과 항목 ({N}개)
| ID | 항목 | 마커 | 비고 |
|:--:|------|:----:|------|

### ❌ 미통과 항목 ({N}개)
| ID | 항목 | 심각도 | 마커 | 상세 | 권장 수정 |
|:--:|------|:------:|:----:|------|----------|

### ⚠️ 부분 통과 ({N}개)
| ID | 항목 | 마커 | 통과율 | 상세 |
|:--:|------|:----:|:------:|------|

### 🔇 검사 불가 ({N}개)
| ID | 항목 | 사유 |
|:--:|------|------|

## 차원 점수
- 통과: {N}/{Total} ({%})
- 심각도 분포: 🔴{N} 🟠{N} 🟡{N} 🔵{N}

## 핵심 인사이트
1. {가장 중요한 발견}
2. {두 번째 발견}
3. {세 번째 발견}
```

### 12.5 Lead 최종 검증 체크리스트

Stage 3에서 Lead가 통합 리포트 생성 전 수행하는 최종 검증:

```
[최종 검증]
□ 모든 TM 리포트 수집 완료 (또는 Fallback 완료)
□ 중복 이슈 제거 완료
□ 15차원 점수 산출 완료
□ 등급 부여 완료
□ 심각도별 정렬 완료
□ 개선 로드맵 생성 완료
□ 환각 마커 없는 발견사항 = 0개 확인
□ [NOT-TESTABLE] 항목 점수 제외 확인
□ 최종 리포트 파일 저장 확인
```
