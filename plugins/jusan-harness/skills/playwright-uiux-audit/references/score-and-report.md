## 10. Stage 3: 리포트 통합 + UX Score 산출

### 10.1 UX Score 산출 (0-100)

Lead가 모든 TM 리포트를 읽고 15차원 가중 점수를 산출한다.

#### 15차원 가중치 체계

| # | 차원 | 가중치 | TM | 산출 기준 |
|:-:|------|:------:|:--:|----------|
| 1 | Typography | 8% | TM1+TM2 | A1+A2 통합 |
| 2 | Layout | 7% | TM3 | B 카테고리 |
| 3 | Accessibility | 12% | TM4+TM5 | C1+C2 통합 (WCAG 가중) |
| 4 | Psychology | 8% | TM6 | D 카테고리 |
| 5 | Animation | 5% | TM7 | E 카테고리 |
| 6 | Interaction | 8% | TM8 | F 카테고리 |
| 7 | IA & Navigation | 7% | TM9 | G 카테고리 |
| 8 | Mobile | 7% | TM10 | H 카테고리 |
| 9 | Visual + i18n | 5% | TM11 | I+Q 통합 |
| 10 | Form UX | 8% | TM12 | M 카테고리 |
| 11 | Design System | 5% | TM13 | J+Edge 통합 |
| 12 | Emotional | 4% | TM14 | K+Edge 통합 |
| 13 | Performance | 5% | TM15 | L+R 통합 |
| 14 | Content | 4% | TM16+TM17 | N+O+Edge 통합 |
| 15 | States | 4% | TM18 | P+Edge 통합 |
| | **합계** | **~102%** | | 반올림 → 100으로 정규화 |

#### 차원별 점수 산출 공식

```
차원_점수 = (통과_항목 / 검사_항목) × 100

검사 불가 항목([NOT-TESTABLE])은 분모에서 제외

UX_총점 = Σ(차원_점수 × 가중치) / Σ(활성_가중치)
         → 미검사 차원의 가중치는 분모에서 제외 후 정규화
```

#### 등급 체계

| 등급 | 점수 | 의미 | 권장 조치 |
|:----:|:----:|------|----------|
| **S** | 90-100 | 탁월한 UX | 유지 + 미세 조정 |
| **A+** | 80-89 | 우수한 UX | 소규모 개선 |
| **A** | 70-79 | 양호한 UX | 중점 영역 개선 |
| **B+** | 60-69 | 평균 이상 UX | 구조적 개선 필요 |
| **B** | 50-59 | 평균 UX | 상당한 개선 필요 |
| **C** | 40-49 | 미흡한 UX | 전면 재설계 고려 |
| **F** | 0-39 | 심각한 UX 문제 | 긴급 개선 필요 |
