## R45: "curl 200 OK = 라이브 작동" 함정

### 발견
- BE 라우트 200 OK 응답 받았지만 실제 페르소나 클릭 시 hydration mismatch / 빈 화면 / 401 산발 발견 가능
- HTTP status는 서버 응답 생존만 확인. UI 작동·페르소나 흐름·데이터 무결성과는 무관

### 본 스킬 적용
- DOM 전수 walk가 진실 — Layer 2 인터랙션
- HTTP status는 보조 시그널만 (Pre-Flight)
- 8축 카오스 #4 (권한)에서 200 OK인데 권한 분기 누락 확인

## R54: "환경 변수 채워짐 = 환경 정상" 함정

### 발견
- `.env.local`에 모든 키가 정의되어 있어도 실제 페르소나가 그 기능을 쓸 수 있는지 별개
- 예: Supabase URL은 정의됐지만 RLS 정책으로 페르소나 권한 없음

### 본 스킬 적용
- Pre-Flight: env 검증 (필요 시)
- STEP ⑤·⑥: 페르소나 walk 자체가 실 사용 검증
- 6 시그널 채널 #3 (라이브러리)에서 라이브러리 설치 ≠ 라이브러리 정상 사용 확인

## R55: "npx playwright test PASS = 라이브 PASS" 함정

### 발견
- 테스트 코드(`*.spec.ts`) 통과 ≠ 실제 페르소나 클릭 통과
- 테스트는 mock·fixture·격리 환경. 라이브는 실 데이터·실 네트워크·실 권한

### 본 스킬 적용
- Playwright **MCP 직접 호출** (테스트 코드 우회)
- `mcp__playwright__browser_*` 도구 = 라이브 페르소나 시뮬
- `playwright.config.ts` 존재하더라도 test runner 미사용

## R76: 자기 위반 함정 (자기 정의 룰을 자기가 어김)

### 발견

- 본문 명시 + 메타 학습 인용은 필요조건이지 충분조건 아님

### 본 스킬 적용

- **DOM 전수 호출** (LLM 임의 판단 0건)

## R-등재 governance (반례·TTL·deprecation·evidence)

본 스킬에서 새 함정 발견 시 R(N+1) 등재 후보. 등재 기준 5 게이트:

| 게이트 | 정의 |
|---|---|
| (a) 다른 도메인 재현 가능 | 본 프로젝트 한정 X |
| (b) 기존 R45-R77과 다른 새 패턴 | 동일 함정 재발견 X |
| (c) "모르면 다시 빠진다" 일반성 | 명시·인용으로 차단 불가 |
| (d) 반례 탐색 | 본 함정이 적용되지 않는 사례 명시 |
| (e) Evidence artifact | 발견 시점 재현 가능 증거 (스크린샷·로그·hash) |

→ 5 게이트 모두 충족 시에만 등재 후보. 부족하면 라운드 리포트에만 남김.

## 메타 학습 lineage 응용 흐름

```
[STEP ② 디스커버리]
     ↓
시그널 채널 #6 (메타 학습) 활성
     ↓
R45/R54/R55/R75/R76/R77 본 프로젝트 적용 가능성 평가
     ↓
적용 가능한 R-등재 → 결함 카테고리 후보로 추가
     ↓
[STEP ③] 카테고리 안에 R-등재 명시 (근거 컬럼)
     ↓
[STEP ⑤·⑥] 해당 카테고리 검수 행동 적용
     ↓
[STEP ⑦] 발견된 결함이 새 R-등재 후보인지 평가
     ↓
사용자 명시 승인 시 → 본 디렉토리에 R(N+1) 추가
```

→ 메타 학습은 살아있는 자산. 단, **R-등재는 사용자 명시 승인** (R-등재 자체가 자체 목적이 되지 않게).
