# 비유 사전 (Analogy Bank) — 기술 → 일상

> **원칙**: 일상에서 자주 경험하는 도메인 (식당/공사장/병원/학교/은행)에서만 비유 추출. 추상 비유 금지.

---

## 도메인 1: 식당 오픈 (기본 — 권장)

| 기술 용어 | 식당 비유 | 강도 |
|---|---|:--:|
| 코드 작업 | 인테리어 + 주방 시설 만들기 | ★★★★★ |
| LAUNCH (오픈일) | 식당 정식 오픈하는 날 | ★★★★★ |
| 결제 (Toss) | 카드 결제기 설치 + 카드사 계약 | ★★★★★ |
| API 키 | 전기·가스·수도·인터넷 가입 (계량기 번호) | ★★★★★ |
| 법적 의무 (PIPA) | 영업 허가증, 보건소 검사 | ★★★★★ |
| 보안 (도어락/금고) | 식당 도어락·CCTV·금고 | ★★★★★ |
| commit (저장) | 메뉴판 인쇄 의뢰 | ★★★★ |
| 데이터베이스 | 주방 냉장고 / 식자재 창고 | ★★★★ |
| 마이그레이션 | 냉장고 정리 + 새 칸 추가 | ★★★ |
| 카테고리 (분류) | 메뉴판 섹션 (한식/양식/디저트) | ★★★★ |
| 사이즈 옵션 | 음료 사이즈 (S/M/L) 또는 토핑 옵션 | ★★★★ |
| 사용자 권한 (RBAC) | 사장/매니저/홀직원/주방장 역할 | ★★★★★ |
| 콘솔 에러 | 주방 알림벨 (피크 시간 경고) | ★★★ |
| 성능 (TTFB) | 주문 후 음식 나오는 시간 | ★★★★ |
| 백엔드 / 프론트엔드 | 주방 / 홀 | ★★★★★ |
| 모달 | 가게 안내문 팝업 (영업 시간 안내 등) | ★★★ |
| 캐시 | 미리 만들어 둔 반찬 | ★★★ |
| 로드 밸런서 | 손님 안내 직원 | ★★★ |
| 알림톡 (카카오) | 주문 확인 전화 | ★★★★ |
| OAuth (소셜 로그인) | 회원증 (다른 카드사 회원증으로 가입) | ★★★ |
| 마이그 production | 영업 시간 중 매장 리모델링 | ★★★ |
| 결함 fix | 메뉴판 오타 수정 | ★★★★ |
| 위양성 (false positive) | 손님 잘못 본 거 (음식 잘못 나왔다 항의 → 실은 정확) | ★★★ |

## 도메인 2: 공사장 / 건축 (대규모 변경)

| 기술 용어 | 공사장 비유 |
|---|---|
| 마이그레이션 (DB schema) | 건물 구조 변경 (벽 허물기) |
| 리팩터링 | 건물 안 가구 재배치 |
| 데이터 손상 | 도면에 잘못 표기된 치수 (14m 짜리 방) |
| 보안 강화 | 도어락 + CCTV + 비상벨 |
| 백업 | 도면 사본 (불 타도 복구 가능) |
| 배포 (deploy) | 입주 청소 후 가구 배치 |
| Rollback | 공사 전 상태로 복원 |

## 도메인 3: 병원 (의료 분야 비유)

| 기술 용어 | 병원 비유 |
|---|---|
| 정밀 audit | 종합검진 |
| 결함 (defect) | 진단된 질병 |
| 보안 incident | 응급 환자 |
| 위양성 | 검진 결과 오진 (실은 건강) |
| 우선순위 (P0~P3) | 응급실 분류 (Triage) |
| Iron Law | 의사 윤리 강령 |
| commit | 진료 기록 차트 작성 |

## 도메인 4: 학교 (교육 시스템 비유)

| 기술 용어 | 학교 비유 |
|---|---|
| 사용자 권한 | 학생/교사/교장 역할 |
| 인증 (JWT) | 학생증 |
| 세션 만료 | 학생증 유효 기간 |
| MFA | 학생증 + 신분증 동시 제시 |
| 페어 프로그래밍 | 공동 과제 |
| 코드 리뷰 | 숙제 채점 |

---

## 도메인 5: 은행 (금융 시스템 비유)

| 기술 용어 | 은행 비유 |
|---|---|
| 결제 (Toss) | 은행 송금 / 계좌이체 |
| 멱등성 (idempotency) | 한 번만 송금 (재시도해도 중복 X) |
| 트랜잭션 | 입금 + 출금 동시 처리 (둘 다 성공 or 둘 다 실패) |
| 잠금 (lock) | 송금 중 계좌 잠금 |
| 동시성 race condition | 동시에 2번 송금 버튼 누른 상태 |
| RLS (Row Level Security) | 다른 사람 계좌 못 보게 차단 |

## 통합 비유 선택 가이드 (Stage 2)

| 주제 | 추천 통합 비유 |
|---|---|
| LAUNCH 준비 / 의제 정리 | **식당 오픈** (기본) |
| 보안 결함 / 취약점 | **식당 도어락/금고** 또는 **은행 RLS** |
| 결제 흐름 | **카드 결제기** 또는 **은행 송금** |
| DB 마이그 / 데이터 변경 | **공사장 도면 변경** |
| audit / 정밀 검토 | **병원 종합검진** |
| 권한/역할 시스템 | **학교 학생증** 또는 **식당 직원 역할** |

**기본값**: 식당 오픈 (사장님 가장 친숙)

## 비유 사용 규칙

1. **카드별 비유 1개 + 통합 비유 1개** 의무 (N+1개)
2. **모호한 추상 비유 금지** — "퍼즐 조각", "톱니바퀴" 같은 비구체 비유 X
3. **사장님 직접 경험 영역**에서 추출 — 사장이 모르는 도메인 (군대, 외국 문화) 사용 X
4. **연결성 강화** — 헤더 비유 → 카드 비유 → 결정 트리 비유 → FAQ 비유 일관 유지
5. **비유 매핑 표** — 핵심 비유 박스에 "기술 ↔ 비유" 표 의무 (사장이 즉시 매핑 확인)
