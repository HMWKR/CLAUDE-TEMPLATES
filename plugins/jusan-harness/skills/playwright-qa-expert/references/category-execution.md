## 15. 카테고리별 실행 (`--category` 옵션)

> **목적**: 175개 전체 항목 대신 특정 영역만 집중 테스트할 때 사용한다.

### 15.1 사용법

```
/playwright-qa-expert --category=<카테고리명>
/playwright-qa-expert --category=<카테고리1>,<카테고리2>
```

### 15.2 카테고리 매핑

| 카테고리명 | 약어 | 체크리스트 항목 | 항목 수 | 설명 |
|-----------|:----:|:-------------:|:-------:|------|
| `typography` | `typo` | #1 ~ #30 | 30 | 폰트 크기, 종류, 행간/자간 |
| `layout` | `lay` | #31 ~ #55 | 25 | 간격, 그리드, 반응형 |
| `color` | `col` | #56 ~ #80 | 25 | 색상 대비, 심리학, 피드백 |
| `psychology` | `psy` | #81 ~ #105 | 25 | 게슈탈트, 인지, 행동 원리 |
| `micro` | `mic` | #106 ~ #125 | 20 | 상태 변화, 애니메이션 |
| `visual` | `vis` | #126 ~ #140 | 15 | 시각적 계층, 일관성 |
| `mobile` | `mob` | #141 ~ #155 | 15 | 터치 타겟, 반응형, 제스처 |
| `edge` | `edg` | #156 ~ #175 | 20 | 엣지케이스, 극한 상황 |
| `a11y` | `a11y` | 접근성 관련 교차 항목 | ~25 | 아래 상세 매핑 참조 |

| Typography | #30 | 텍스트 잘림 ellipsis + title |
| Color | #56 | 본문 텍스트 대비 4.5:1 |
| Color | #57 | 대형 텍스트 대비 3:1 |
| Color | #58 | 비텍스트 요소 대비 3:1 |
| Color | #59 | 플레이스홀더 대비 |
| Color | #60 | 비활성 요소 대비 |
| Color | #61 | 포커스 표시 대비 3:1 |

### 15.4 복수 카테고리 실행 예시

```bash
# 접근성만 테스트
/playwright-qa-expert --category=a11y

# 모바일 + 접근성 동시 테스트
/playwright-qa-expert --category=mobile,a11y

# 타이포그래피 + 색상 테스트
/playwright-qa-expert --category=typo,col

# 약어도 전체 이름도 사용 가능
/playwright-qa-expert --category=typography,color
```

### 15.5 카테고리 모드 실행 규칙

1. **티어 무시**: `--category` 사용 시 티어 구분 없이 해당 카테고리의 **모든 항목** 테스트
2. **전문가 선택**: 카테고리에 가장 적합한 전문가가 자동 선택됨
3. **리포트**: 카테고리별 집중 리포트 생성 (전체 리포트보다 간소화)
4. **조합 가능**: `--full`과 조합 불가 (카테고리 모드는 독립 실행)

| 카테고리 | 주요 담당 전문가 |
|---------|----------------|
| `typography` | UX 디자이너, 프론트엔드 아키텍트 |
| `layout` | 프론트엔드 아키텍트, 모바일 UX 전문가 |
| `color` | 접근성 전문가, UX 디자이너 |
| `psychology` | 사용자 심리학자 |
| `micro` | 프론트엔드 아키텍트, UX 디자이너 |
| `visual` | UX 디자이너, 프론트엔드 아키텍트 |
| `mobile` | 모바일 UX 전문가 |
| `edge` | 프론트엔드 아키텍트, QA 엔지니어 |
| `a11y` | 접근성 전문가 |
