# 사업계획서 + 인프라 규칙 (STAGE 7 상세)

## 사업계획서 (executive-summary.md)

### 필수 섹션
1. **문제 정의**: 현재 시장의 해결되지 않은 문제 (2~3개)
2. **솔루션**: 이 프로젝트가 어떻게 해결하는가
3. **시장**: TAM/SAM/SOM + 성장률. 수치에 반드시 [추정] 또는 [검색확인] 태그
4. **비즈니스 모델**: 수익원, 단가, 과금 방식
5. **경쟁 우위**: 기존 대비 차별화 (비교 테이블)
6. **Go-to-Market**: Phase 1/2/3 로드맵 + 마케팅 채널
7. **재무 계획**: 초기 투자 소요 테이블 + BEP 시나리오 + KPI 목표
8. **팀**: MVP/Phase2/Phase3 필요 인원
9. **리스크**: 실패 가능 이유 3가지 + 대응 전략
10. **자금 요청**: 시드 규모 + 용도 비율 + 마일스톤

### 수치 규칙
- 시장 규모: 웹 검색으로 확인 가능하면 검색. 불가하면 [추정] 태그
- 비용: 인프라(AWS 기준 구체적 서비스별), 인건비(직군별 시장 단가), 마케팅(CAC 기반)
- 매출 예측: 보수적 시나리오 기준, MAU → 전환율 → AOV로 산출

## 인프라 설계 (aws-architecture.md)

### 필수 포함
1. 서비스별 AWS 리소스 매핑
2. MAU 구간별(1만/5만/20만) 월 비용 추정 테이블
3. CI/CD 파이프라인 (GitHub Actions YAML 예시)
4. 오토스케일링 전략
5. 모니터링 스택

## CLI 사용 가이드 (CLI-SETUP-GUIDE.md)

### 필수 포함
1. 초기 세팅 (ZIP 해제 → Git 초기화 → Claude Code 실행)
2. 도메인 날리지 주입 방법
3. 에이전트 활용법 (로드, 직접 호출, 크로스 부서, 부서 단위)
4. 실전 개발 프롬프트 예시 (최소 20개)
   - 프로젝트 초기화
   - 핵심 기능 구현
   - DB/API 구현
   - 코드 리뷰
   - 배포
   - 장애 대응
   - 새 기능 추가
5. 트러블슈팅 (CLAUDE.md 미인식, 에이전트 컨텍스트 유실 등)
