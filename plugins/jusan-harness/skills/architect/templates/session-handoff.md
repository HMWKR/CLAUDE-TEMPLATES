## 작성 원칙

1. **현재 상태 정확히 반영**: 완료/진행중/미구현을 명확히 구분
2. **다음 세션에서 즉시 시작 가능**: 어디서부터 이어야 하는지 명확
3. **GAP 추적**: 설계 vs 구현 차이를 지속 업데이트
4. **파일 인벤토리**: 생성된 파일과 예정된 파일 모두 기록
5. **핵심 결정 사항**: 왜 그렇게 결정했는지 근거 포함 (다음 세션에서 재논의 방지)
