# Presentation Engine — 범용 발표 덱 제작 (맥락 연역 + 프로젝트-네이티브 스토리)

> **전체 방법론 SSoT**: 별도 방법론 문서(`PPT-제작-방법론.md`)가 **있으면** 참조(이 스킬은 그 실행기, 깊이 필요 시 Read). 없으면 이 SKILL.md 본문만으로 자족 실행.
> **골든 레퍼런스(읽기 전용 *사례*)**: 골든 레퍼런스 덱이 **있으면** — **구조/CSS/컴포넌트/빌드만 복제, 텍스트·spine·카피·수치는 0에서**.
> **출력**: `<프로젝트명> 발표자료/<YYYY-MM-DD>/` (작업 cwd 기준). 슬라이드 우선(HTML→PNG→PPTX), 대본·근거·배포본 선택. 한국어 기본+영어 대응.

## 세 개의 북극성 (충돌 시 우선)
1. **맥락이 슬라이드를 결정한다.**
2. **청중이 내려야 할 결정에 필요한 것만 준다** — 발표자 자랑 순서가 아니라 듣는 사람의 결정 순서.
3. **레퍼런스는 *생성 원리*만 가져오고 문구·spine·예시는 0에서 새로** — 복제하면 다른 맥락에서 틀린 덱이 된다.
> ⚠️ "약속이 아니라 증거"는 보편 법칙이 **아니라** MongPet 맥락이 도출한 *그 프로젝트 spine 사례*. 각 프로젝트는 자기 spine을 새로 만든다.

## 실행 파이프라인 (게이트 통과 못 하면 다음 단계 금지)

```
S0 맥락(코어8+프로파일)  →[게이트 코어8 공란0]→ 👤확인
S1 Soul-to-Spine         →[게이트 spine 생성+복제토큰0]→ 👤 spine 확정
S2 청중 정렬(WIIFM)       →[게이트 전 슬라이드 takeaway↔청중needs 1:1]
S3 프레임워크 선택+아크    →[게이트 메뉴서 1개 선택 근거]
S4 한 장 공식            →
S5 증거 패스(+viz정합·접근성) → S5.5 반론·Q&A
S6 빌드(HTML→PNG→PPTX)   →
S7 QA 루프(7차원·4페르소나) →[통과까지 반복]→ 👤 종결 → 출력
   FAIL: 청중매핑→S2 / 복제·spine→S1 / 증거부족→S5 / 과밀→S4
```
**사용자 확인 필수(👤)**: S0 종료·S1 spine 확정·S7 종결. 되돌아가기 단계별 최대 3회.

## S0 — 맥락 코어 8 (모르면 AskUserQuestion, 임의판단 금지)
①청중이 내려야 할 결정 ②청중 정체성·사전지식(0~3) ③청중 WIIFM·위험 ④신뢰격차(의심 Top1~3) ⑤발표자 목표 ⑥시간·형식·채널 ⑦성공 정의 ⑧도메인 제약(규제·컴플라이언스·인증·보안 — 핀테크/헬스/리걸 등에선 spine 좌우, 펫·일반은 "해당없음" 명시).
그다음 **프로파일 1개 선택**: 심사·투자(IR, 한국정부는 서브모듈 PSST·자금·정책·가점) / 세일즈 / 내부보고 / 제품출시 / 교육·강연 / 컨퍼런스. 프로파일은 골격·게이트만 주고 spine은 안 준다.

## S1 — Soul-to-Spine ⭐ (복제 방지 핵심)
(a) **고유 자산 1개** 식별(7범주: 실물완성도/독점데이터·기술/창업자서사/도메인통찰/네트워크/사회적임팩트/**규제·라이선스·인증 선점** → 주자산 1개 수렴, 복합이면 +보강 1개) → (b) **청중이 YES하려면 믿어야 할 것 1문장**(코어0·3 역산) → (c) **spine 1문장 합성**: "(고유자산)이기에 (청중신념)이 가능하다"를 *이 프로젝트 고유 단어*로 → (d) **S1-4 금칙어 추출**: 참조 레퍼런스(들)의 *실제 텍스트*를 Read해 그 덱의 spine·슬로건·예시수치를 `_blacklist.txt`로 생성(고정 목록 아님·프로젝트별).
**복제 금지 게이트**: G1(grep — `_blacklist.txt` 토큰 0건, 결정적) + G2(의미 — 대구슬로건·절대표현·타도메인 상징어·예시수치·컴포넌트조합 이식 여부, blacklist 비어도 작동) + G3(spine이 (a)자산서 도출·청중신념에 닿나). 1건이라도 FAIL→S1 재실행(👤). spine은 표지·중간·클로징 3회(이 프로젝트의 문장으로).

## S2 — 청중 정렬
WIIFM 번역: 각 명제 → 청중 성공지표/이득 언어로 1:1(번역 불가=자기자랑→삭제). 모든 슬라이드 takeaway가 청중 needs에 매핑되는지 게이트. 첫 슬라이드는 청중의 현재 현실/감정에서 출발.

## S3 — 프레임워크 메뉴(맥락에 맞는 1개 선택)
SCQA(논리·내부·임원) / Monroe's(행동촉구·영업·기금) / Pixar Spine(감정·소셜·창업자·고객사례) / Hero's Journey(교육·변화) / Evidence-Climax(고의심·실물형·심사=MongPet사례). 공통: 한 장 한 비트+접속사+spine 3회. 자가점검 Heath SUCCESs.

## S4 — 한 장 공식
`kicker + 주장형 제목(action title, 명사라벨 X) + 1컴포넌트 + takeaway`. 한 장 한 메시지.
시각: 작업기억4±·신호잡음·이중부호화 / hero number는 스탯장만 / 60-30-10·강조색10%(CSS토큰) / 신뢰자산 회색처리 금지(2단 위계). 데이터viz 정합(축절단 금지·차트1개=주장1개). 접근성(명암비4.5:1·색맹안전·그레이스케일). 슬라이드=신호, 대본=내용.

## S5 — 증거 + S5.5 반론·Q&A
수치↔출처 1:1, 추정 표기, 근거 별도 문서, 1차출처 우선. 프로파일별 증거 어휘(IR=작동·재무 / 세일즈=고객케이스·ROI / 내부=파일럿·비용편익 / 교육=실증·사례). traction 부재 시 "베이스라인(업계평균)→목표→측정방법". 의심 Top3 → 슬라이드로 선제 무력화 + ARA(Acknowledge-Reframe-Answer) Q&A 노트.

## S6 — 빌드
골든 레퍼런스 HTML 구조/CSS토큰/빌드 복제(텍스트는 0에서). 1280×720 #stage → 브라우저(우선순위: rules/uncompromising-rigor §1, Playwright MCP 전역 우선)로 #stage 캡처(DSF2x, export 모드) → python build_pptx.py → 16:9 PPTX.

## S7 — QA 루프 (통과까지 반복) ⭐
글로벌 `~/.claude/rules/uncompromising-rigor.md` 준수(모든 발견=결함·자기정당화 차단).
- **자동 게이트(기계 판정)**: G1 복제 grep(`_blacklist.txt` 0건) / G4 시각·접근성 측정(렌더=HTML 일치·명암비≥4.5:1·색맹/그레이스케일·강조색≤10%) / G5 정합(수치↔출처 1:1·장수≤상한·HTML=PPTX·낭독시간≤시간상한).
- **의미 게이트(LLM)**: G2 복제 의미판정(부록B-1 추상패턴) / G3 spine 정합(자산 도출·청중신념).
- **루브릭 7차원**(총점=Σ 차원점수/5×가중): 청중적합20·맥락정합15·서사응집15·증거정직20·인지부하10·접근성10·프로젝트네이티브독창성10.
- **4 페르소나**: (a)타깃청중 대역 — **청중 5문항**(이득 즉시 인지/3장째까지 이해/의심Top3 해소/결정기준 충족/끝에 행동 명확) (b)적대적 red-team(ARA 방어) (c)인지·접근성 검수 (d)레퍼런스-오염 감별사.
- **통과**: G1·G2·G3·G4·G5 PASS AND 총점≥85 AND 전차원≥3 AND **critical(청중·증거정직·접근성·독창성)≥4** AND 청중 5문항 전부 통과 AND red-team 미방어0.
- **FAIL 되돌아가기**: 청중/5문항→S2·맥락정합→S0/S2·서사/독창성/복제→S1·증거→S5·red-team→S5.5·과밀→S4·접근성/시각→S4·빌드/타이밍→S6.
- **반복 가드**: 외부 QA 라운드 **최대 3회**(내부 되돌아가기도 3회). 미통과 시 잔여결함 리포트+사용자 에스컬레이션(임의 통과·종료 금지). 👤 종결 승인으로만 종료.
> Agent Teams(AGENT_TEAMS=1) 가능 시 4 페르소나 병렬, 아니면 순차 self-critique.

## 패키지
필수: 발표자료.html·PPTX·slides/·screenshots/. 선택: 대본(2분/1분/팁)·데이터-근거.md·배포본(요약+Q&A응답서+다음단계)·슬라이드 노트(말할내용/소요초/예상질문ARA/전환/출처)·리허설 시간측정.

## 안티패턴 (복제 금지 — 고정 리터럴 아닌 *추상 패턴*으로)
- 레퍼런스의 spine·슬로건·상징 카피를 다른 프로젝트로 이식(어떤 리터럴이든). "A가 아니라 B" 대구·"국내 최초 X"류 절대표현·특정 도메인 상징어를 그대로.
- 레퍼런스 예시 수치·도메인 사례 그대로 끌어오기(타 프로젝트 수치 ≠ 이 프로젝트 증거).
- **컴포넌트 조합·비중 복제**(실물형 레퍼런스의 스크린샷그리드 중심 구성을 무형 SaaS가 그대로) → S4에서 이 프로젝트 증거유형에 맞게 0에서 재선택.
- "복제해 시작" = 구조/CSS/컴포넌트클래스/빌드만, 텍스트·spine·카피·수치·조합비중은 0에서.
- (시각) 모든 슬라이드 거대 hero number / 신뢰자산 회색처리 / 단정 법조항·배점·환각 수치 / 코드 노골 스탬프 / 추정 수치 무라벨.
> 구체 금칙어는 S1-4에서 *참조 레퍼런스 실제 텍스트*를 Read해 프로젝트별 `_blacklist.txt`로 생성(고정 목록 금지 — 신규 도메인엔 무력).

## 출력 위치 규약
`<프로젝트명> 발표자료/<YYYY-MM-DD>/` (작업 cwd). 분석 대상 프로젝트 디렉토리에는 **산출물을 만들지 않는다(읽기 전용)** — 항상 작업 디렉토리에 생성.
