# 프롬프트 저널 템플릿

이 템플릿을 사용하여 프롬프트와 사고 과정을 기록하세요.

## 사용법

1. `.prompts/` 폴더에 새 파일 생성
2. 파일명 형식: `YYYY-MM-DD-{topic}.md`
3. 아래 템플릿을 복사하여 작성

---

## 템플릿

```markdown
---
# 메타데이터 (YAML frontmatter)
date: YYYY-MM-DD
time: "HH:MM"
commit: null  # 커밋 후 해시 추가
branch: main
domain: general  # backend, frontend, devops, docs, etc.
subdomain: null  # auth, api, ui, etc.
complexity: medium  # low, medium, high
quality_score: null  # 평가 후 추가 (0-48)
grade: null  # S, A+, A, B+, B, C, F
tags: []  # 관련 키워드
---

# [제목: 작업 내용]

## 원본 프롬프트

> 사용자가 입력한 프롬프트를 그대로 기록

## 사고 여정

### 처음 생각
처음에 어떻게 접근하려고 했는가?

### 전환점
무엇이 생각을 바꾸게 했는가?
- 전환점 1: ...
- 전환점 2: ...

### 최종 통찰
결국 어떤 결론에 도달했는가?

## 최적화된 프롬프트

```
개선된 프롬프트를 여기에 작성
- 명확한 요구사항
- 제약조건
- 검증 방법

## YAML frontmatter 스키마

| 필드 | 타입 | 필수 | 설명 | 예시 |
|------|------|:----:|------|------|
| `date` | string | O | 작성 날짜 | `2026-01-14` |
| `time` | string | X | 작성 시간 | `"14:30"` |
| `commit` | string | X | 관련 커밋 해시 | `abc1234` |
| `branch` | string | X | 브랜치명 | `feature/auth` |
| `domain` | string | O | 주요 도메인 | `backend` |
| `subdomain` | string | X | 세부 도메인 | `authentication` |
| `complexity` | string | O | 복잡도 | `low`, `medium`, `high` |
| `quality_score` | number | X | 품질 점수 (0-48) | `42` |
| `grade` | string | X | 등급 | `S`, `A+`, `A`, `B+`, `B`, `C`, `F` |
| `tags` | array | X | 태그 목록 | `[JWT, security]` |

### domain 값 목록

| domain | 설명 |
|--------|------|
| `backend` | 서버, API, 데이터베이스 |
| `frontend` | UI, 클라이언트, 웹 |
| `devops` | CI/CD, 배포, 인프라 |
| `docs` | 문서, README |
| `test` | 테스트, QA |
| `refactor` | 리팩토링 |
| `security` | 보안 |
| `performance` | 성능 최적화 |
| `general` | 기타 |

## 섹션 설명

### 원본 프롬프트
사용자가 Claude에게 입력한 프롬프트를 **그대로** 기록합니다.
나중에 "원래 어떻게 요청했는지" 참고할 수 있습니다.

### 사고 여정
Claude(또는 사용자)의 사고 과정을 기록합니다:
- **처음 생각**: 최초의 접근 방식
- **전환점**: 생각이 바뀐 계기
- **최종 통찰**: 결론에 도달한 이유

이 섹션은 "왜 이렇게 했는가"를 이해하는 데 핵심입니다.

### 최적화된 프롬프트
원본 프롬프트를 개선한 버전입니다.
다음 요소를 포함하면 좋습니다:
- 명확한 요구사항
- 제약조건
- 검증 방법
- 기대 산출물

### 품질 분석
5계층 48점 시스템으로 프롬프트 품질을 평가합니다.

### 핵심 학습
이번 작업에서 배운 점을 정리합니다.
나중에 비슷한 상황에서 참고할 수 있습니다.

### 결과
작업 결과를 요약합니다.
- 성공/실패 여부
- 반복 횟수 (몇 번 시도했는가)
- 변경된 파일 수
- 테스트 통과 여부
