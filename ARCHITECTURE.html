01 — Architecture

5계층 (L0 → L4)

각 계층은 모델의 약점을 다른 수단으로 보상한다. 아래로 갈수록 "누가 일하나"에서 "어떻게 통제하나"로.

L0 어댑터모델 적응
세션 모델 감지 → 처방 프로파일 주입. transcript 파싱으로 모델 판별, 미검출=고처방 안전폴백.
adapters/{fable,opus,sonnet,_default}.md · adapter-inject.sh (UserPromptSubmit)
L1 증폭인지 = 토큰
판단 깊이를 표본 수 N으로 구매. 판정단·렌즈분산·자기일관성·성찰·적대검증 + 이종(교차벤더) 판정단.
skills/amplify-presets (태스크별 N표 · 이종 판정단 · 손익 백오프)
L2 게이트검증 주권
증거 없는 완료 차단 + 검증 주권 분리. DoD·채점은 실행 모델 밖. tamper 해시·카나리.
fablize gate_stop/gate_post_tool · verification-templates/ · verify-lock.py · emulation-guard(관찰)
L3 상태·기억영속
세션·압축 경계 넘어 상태 영속. 파일=기억, PreCompact flush로 압축 후 연속성.
checkpoint.md · checkpoint-save.py · 파일 메모리 · tasks/<task>/{task,log,brief,result}.md
L4 오케스트레이터제어=코드
제어 흐름을 코드가 소유. 성공측 종료(신규발견0×2·DoD·카나리) + 실패측 발산차단기(K회→에스컬레이션).
네이티브 Workflow/Agent teams · conductor-verify.js · goals.py(/goal)

횡단 원칙 — 검증 주권 분리: "무엇을 검증할지"의 저작·채점을 실행 모델 밖에 둔다. 안 그러면 "모델이 약할수록 더 빨리·더 확신 있게 잘못된 완료 선언"하는 목적 역전.

03 — Orchestration
