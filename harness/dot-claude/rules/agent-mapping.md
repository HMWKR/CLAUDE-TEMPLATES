# 에이전트 페르소나 가이드

> 2026-06-11 재작성: 강제 게이트(30분 타임스탬프·방법론 사이클) 제거. 구버전: `~/.claude/backups/pre-ultimate-2026-06-11/rules/`

## 원칙

- 에이전트 정의(`.claude/agents/<name>.md`)는 **품질을 위한 페르소나 자산**이다. 도메인 작업 시 맞는 정의를 읽고 그 역할·도구 권한·출력 형식으로 작업한다.
- 매핑(메인이 직접 변신) vs Spawn(Agent 도구) 선택 기준:

| 상황 | 권장 |
|---|---|
| 단일 도메인 일상 작업 | 매핑 (메인 컨텍스트 활용) |
| 대규모 탐색·일괄 생성·긴 리서치 회수 | Spawn (컨텍스트 격리) |
| 독립 검증·교차 검토 | Spawn |

- 페르소나의 도구 권한 범위를 넘는 도구 사용은 자제한다 (예: 리뷰어 페르소나일 때 Write 자제).
- 작업 중 발견·결정이 재사용 가치가 있으면 프로젝트 인사이트 파일(`docs/domain-knowledge/`)에 기록한다 — 의무가 아니라 자산화.
