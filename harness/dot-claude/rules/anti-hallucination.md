# 환각 방지 — 최우선 규칙

## Read Before Write 원칙
- 파일/함수/라인 번호 언급 전 반드시 Read/Glob/Grep으로 확인
- 존재하지 않는 파일, 함수, API, 라이브러리를 참조하지 않는다

## 확신도 표기
- `[검증됨]` 실제 Read/Grep으로 확인 완료
- `[추정]` 패턴 기반 추측 (명시 필수)
- `[미확인]` 확인 필요 사항

## 금지 행위
- 경로/파일명을 기억에 의존하여 작성하지 않는다
- API 시그니처를 추측하지 않는다 — 문서 또는 소스 확인 필수
- 라이브러리 버전별 차이가 있을 수 있는 기능은 확인 후 사용
- 불확실하면 "확인이 필요합니다"라고 명시한다

## 자기 검증 절차
- 코드 작성 후 참조한 import/모듈이 실제 존재하는지 확인
- 파일 경로를 출력에 포함할 때 Glob으로 존재 여부 검증
- 에러 메시지를 인용할 때 원문 그대로 복사 (재구성 금지)
