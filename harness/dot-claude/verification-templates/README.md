# 검증 템플릿 라이브러리 (청사진 v2 N5 — 검증 주권 분리)

> 원칙 4: DoD·verify-cmd의 저작·채점은 피검(실행) 모델 밖에 둔다.
> 실행 모델이 스스로 시험지를 쓰고 채점하면 게이트는 형식 검사로 퇴화한다.

## 사용 규약

1. **플랜 시점**: 태스크 유형에 맞는 템플릿을 선택하고 `<>` 자리만 구체화한다. 항목 삭제 금지(해당 없으면 N/A + 사유).
2. **커버리지 밖 태스크**: 템플릿이 맞지 않으면 자기 저작 대신 이종/상위 모델 판정단의 승인을 받는다.
3. **해시 고정**: 확정 즉시 `python3 ~/.claude/scripts/verify-lock.py lock <작업디렉토리>/.verify-lock.json <DoD파일> <테스트파일...>`.
4. **실행 중**: 실행 노드는 DoD·verify-cmd·잠긴 테스트를 수정할 수 없다. 게이트 실패 시 검증 기준이 아니라 접근을 바꾼다.
5. **완료 판정 전**: `verify-lock.py check` — TAMPER 검출 시 게이트 무효, 재승인 필요.
6. **카나리**: review-qa 템플릿의 카나리 항목은 "신규 발견 0" 수렴의 유효성 전제다 — 탐지율 임계 미달이면 수렴 무효.
