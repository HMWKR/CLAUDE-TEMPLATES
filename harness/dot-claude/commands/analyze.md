# Deep Analysis Mode

> 철학: 얕은 표면 PASS 금지 — 증거 기반·다관점.

## 현행 하네스 규율 적용
1. **plan-first**: 실행 전 계획(분해 + DoD), 다단계면 투두(TaskCreate)로 추적.
2. **conductor-verify 구조**: 계획 → 검수 → 실행 → 종합 → 독립 검증 → 승인. "판단은 main, 검증증거는 밖."
3. **환각 방지**(엄격): 파일·함수·경로는 Read/Grep 확인 후 언급.
4. **investigation 신호 시**(디버그·원인불명·리뷰): 재현 → 3+ 경쟁 가설 → 가설별 증거 → 인과사슬 → 전후 검증 → 기각 가설 보고.
5. **검증 마커**: `[검증됨]`(확인) · `[추정]`(패턴) · `[미확인]`(확인필요).
6. **완료**: 실행 증거 + 게이트 통과. 코드만 읽고 OK 하는 표면 PASS 금지.

## 5단계 메타인지 자가 검증
```
1. 가정 검토: 당연하게 여기는 전제는 무엇인가?
2. 반대 관점: 이 결론에 반대하는 가장 강력한 논거는?
3. 실패 모드: 이 해결책이 실패할 수 있는 시나리오는?
4. 숨겨진 변수: 내가 놓치고 있을 수 있는 요소는?
5. 확신도: 내 결론에 대한 확신 (1-10) + 근거.
```

깊이 분석하고, 증거로 검증하며 진행하세요.
