# PDCA Gap 검증 모드 활성화

> 대상 기능: $ARGUMENTS (미지정 시 현재 작업 대상 자동 감지)

## 1단계: 설계문서 탐색

프로젝트 내 설계 관련 문서를 탐색한다:
- `docs/`, `design/`, `spec/`, `*.design.md`, `*.spec.md`, `README.md` 내 설계 섹션
- 설계문서 미발견 시: "설계문서가 없습니다. 비교 대상을 지정해주세요." → 중단

## 2단계: 7카테고리 Gap 비교

설계문서와 실제 구현을 다음 7개 축으로 비교한다:

| # | 카테고리 | 비교 대상 |
|:-:|----------|----------|
| 1 | API 엔드포인트 | URL, HTTP 메서드, 요청/응답 형식 |
| 2 | 데이터 모델 | 엔티티, 필드, 관계, 제약조건 |
| 3 | 핵심 기능 | 설계된 기능 vs 구현된 기능 |
| 4 | UI/UX 흐름 | 화면 구성, 사용자 흐름 |
| 5 | 환경 변수 | 필요 설정 vs 실제 설정 |
| 6 | 아키텍처 | 레이어 구조, 의존성 방향 |
| 7 | 코딩 규칙 | 네이밍, 폴더 구조, 임포트 패턴 |

각 카테고리별 일치/불일치 항목을 표로 출력한다.

## 3단계: Match Rate 산출

- 전체 비교 항목 중 일치 비율(%) 산출
- 불일치 항목을 Missing(미구현) / Added(설계에 없는 추가) / Changed(변경됨) 분류

## 4단계: 반복 개선 (Match Rate < 90% 시)

- 불일치 항목을 우선순위(영향도) 기준 정렬
- 상위 항목부터 수정 제안 → 사용자 승인 → 구현
- 수정 후 재비교 (최대 3회 반복)
- 90% 도달 또는 3회 완료 시 종료

## 5단계: 결과 보고

```
📊 PDCA Gap 분석 결과
├── 대상: [기능명]
├── Match Rate: [N]% ([이전] → [현재])
├── 비교 항목: [N]개
├── 일치: [N] | Missing: [N] | Added: [N] | Changed: [N]
└── 반복 횟수: [N]/3
```
