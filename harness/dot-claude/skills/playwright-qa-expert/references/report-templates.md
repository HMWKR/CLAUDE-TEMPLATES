## 14. 리포트 비교 기능

> **목적**: 이전 테스트 결과와 비교하여 개선/악화 추이를 추적한다.

### 14.1 리포트 파일명 규칙

```
qa-report-{프로젝트명}-{YYYY-MM-DD-HH-MM}.md
```

**예시**:
```
qa-report-prompt-vault-2026-02-07-14-30.md
qa-report-my-app-2026-02-07-16-45.md
```

**저장 위치**: 프로젝트 루트의 `qa-reports/` 폴더

```
프로젝트-root/
└── qa-reports/
    ├── qa-report-prompt-vault-2026-02-01-10-00.md   (이전)
    ├── qa-report-prompt-vault-2026-02-07-14-30.md   (최신)
    └── ...
```

### 14.2 리포트 비교 섹션 템플릿

최신 리포트 생성 시, `qa-reports/` 폴더에 이전 리포트가 존재하면 자동으로 비교 섹션을 추가한다.

```markdown
## 📊 이전 리포트 대비 변화

### 비교 기준
| 항목 | 이전 리포트 | 현재 리포트 |
|------|-----------|-----------|
| **파일명** | qa-report-xxx-2026-02-01-10-00.md | qa-report-xxx-2026-02-07-14-30.md |
| **테스트 모드** | 기본 (Tier 1) | 전체 (Tier 1+2) |
| **전문가 수** | 6명 | 7명 (+i18n 전문가) |

### 이슈 변화 요약
| 심각도 | 이전 | 현재 | 변화 |
|:------:|:----:|:----:|:----:|
| 🔴 Critical | 3 | 1 | ✅ -2 (개선) |
| 🟠 Major | 5 | 4 | ✅ -1 (개선) |
| 🟡 Minor | 8 | 10 | ⚠️ +2 (증가) |
| 🔵 Suggestion | 12 | 15 | 📝 +3 (신규 제안) |

### 해결된 이슈 ✅
| # | 이전 이슈 | 심각도 | 해결 확인 |
|:-:|----------|:------:|----------|
| 1 | ISSUE-001: 모바일 터치 타겟 미달 | 🔴 | [스크린샷] 44x44px 이상 확인 |
| 2 | ISSUE-003: Tab 순서 비논리적 | 🟠 | [직접테스트] 논리적 순서 확인 |

### 신규 발견 이슈 🆕
| # | 이슈 | 심각도 | 발견 전문가 |
|:-:|------|:------:|------------|
| 1 | ISSUE-008: 다크모드 대비 부족 | 🟡 | 접근성 전문가 |

### 악화된 이슈 ⚠️
| # | 이슈 | 이전 심각도 | 현재 심각도 | 비고 |
|:-:|------|:----------:|:----------:|------|
| - | (악화 항목 없음) | - | - | - |

### 개선 추이 그래프 (텍스트)
```
이전 (02-01): 🔴🔴🔴 🟠🟠🟠🟠🟠 🟡🟡🟡🟡🟡🟡🟡🟡
현재 (02-07): 🔴     🟠🟠🟠🟠   🟡🟡🟡🟡🟡🟡🟡🟡🟡🟡
              ──────────────────────────────────────────
              Critical -67%  Major -20%  Minor +25%
```
```

### 14.3 비교 프로세스

```
1. 리포트 생성 시작
2. qa-reports/ 폴더 존재 확인
   ├─ 없으면 → 폴더 생성 + 비교 섹션 없이 첫 리포트 생성
   └─ 있으면 → 가장 최근 리포트 파일 읽기
3. 이전 리포트에서 이슈 목록 추출
4. 현재 테스트 결과와 이슈 ID/설명 매칭
5. 해결/신규/악화/유지 분류
6. 비교 섹션 생성 후 리포트에 포함
7. 리포트 파일 저장
```
