## 13. 에러 핸들링 & 재시도 가이드

> **원칙**: Playwright 도구 실패는 테스트 실패가 아니다. 도구 실패 시 대체 수단을 활용하고, 검증 불가 항목은 `[검증불가]` 마커로 정직하게 표시한다.

### 13.1 도구별 실패 대응 전략

| Playwright 도구 | 실패 원인 | 1차 대응 (재시도) | 2차 대응 (대체) | 3차 대응 (스킵) |
|----------------|----------|------------------|----------------|----------------|
| `browser_navigate` | 타임아웃, URL 오류 | 5초 대기 후 재시도 (1회) | URL 확인 후 수정하여 재시도 | `[검증불가]` 표시 + 리포트에 기록 |
| `browser_snapshot` | 페이지 미로드 | `browser_wait_for` 후 재시도 | `browser_take_screenshot`으로 대체 | 스크린샷 기반으로 시각 분석 |
| `browser_take_screenshot` | 렌더링 실패 | 2초 대기 후 재시도 | `browser_snapshot` 접근성 트리로 대체 | `[검증불가]` 표시 |
| `browser_click` | 요소 없음, 가려짐 | `browser_wait_for` 후 재시도 | `browser_evaluate`로 직접 클릭 | 이슈로 기록 (요소 접근 불가) |
| `browser_evaluate` | JS 에러, 셀렉터 없음 | 셀렉터 수정 후 재시도 | `browser_snapshot`으로 구조 확인 후 셀렉터 재작성 | 수동 스냅샷 분석으로 대체 |
| `browser_type` | 입력 필드 없음 | `browser_snapshot`으로 필드 확인 후 재시도 | `browser_fill_form`으로 대체 | `[검증불가]` 표시 |
| `browser_hover` | 요소 접근 불가 | `browser_wait_for` 후 재시도 | `browser_evaluate`로 :hover 스타일 직접 확인 | 호버 상태 검증 스킵 |
| `browser_resize` | 뷰포트 변경 실패 | 즉시 재시도 (1회) | 다른 해상도로 시도 | 현재 뷰포트에서 테스트 계속 |
| `browser_press_key` | 키 이벤트 미전달 | 포커스 확인 후 재시도 | `browser_evaluate`로 키 이벤트 디스패치 | `[검증불가]` 표시 |
| `browser_console_messages` | 메시지 없음 | 페이지 새로고침 후 재수집 | 정상 상태로 판단 (에러 0건) | - |
| `browser_network_requests` | 수집 실패 | 페이지 새로고침 후 재수집 | 콘솔 메시지로 네트워크 에러 간접 확인 | `[검증불가]` 표시 |

### 13.2 재시도 프로토콜

```
실패 감지 → [1차] 동일 도구 재시도 (최대 1회, 2-5초 대기)
         → [실패] → [2차] 대체 도구 사용
         → [실패] → [3차] 해당 항목 [검증불가] 표시
         → 다음 항목으로 진행 (테스트 중단 금지)
```

**핵심 규칙**:
- 단일 항목 실패로 **전체 테스트를 중단하지 않는다**
- 재시도는 **최대 1회**만 (무한 루프 방지)
- 대체 도구 사용 시 **리포트에 대체 수단 명시**
- 검증 불가 항목은 **별도 섹션에 모아서 보고**

### 13.3 Graceful Degradation 전략

| 상황 | 영향 | 대응 |
|------|------|------|
| 페이지 로드 완전 실패 | 전체 테스트 불가 | URL 확인 요청 + 테스트 종료 |
| 특정 페이지만 실패 | 해당 페이지 테스트 불가 | 다른 페이지 계속 + 실패 페이지 보고 |
| JavaScript 비활성 | evaluate 스니펫 실행 불가 | 스냅샷/스크린샷 기반 시각 분석만 수행 |
| 느린 네트워크 | 타임아웃 빈번 | 대기 시간 늘리고 핵심 항목(Tier 1)만 테스트 |
| 인증 필요 페이지 | 로그인 이후 콘텐츠 접근 불가 | 사용자에게 로그인 요청 후 계속 |

### 13.4 검증 불가 항목 리포트 형식

```markdown
## ⚠️ 검증 불가 항목

| # | 체크리스트 항목 | 실패 도구 | 원인 | 대체 시도 | 권장 수동 확인 |
|:-:|---------------|----------|------|----------|--------------|
| 1 | #56 색상 대비 | browser_evaluate | JS 에러 | 스냅샷 시각 분석 | 개발자 도구에서 대비 확인 |
| 2 | #141 터치 타겟 | browser_evaluate | 셀렉터 불일치 | - | 모바일 기기 실제 테스트 |
```
