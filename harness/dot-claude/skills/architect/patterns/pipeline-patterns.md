# 파이프라인 아키텍처 패턴 참조

---

## 1. ETL / ELT (Extract-Transform-Load)

### ETL (전통적)

**구조**:
```
[Extract] ──▶ [Transform] ──▶ [Load]
 소스에서     변환/정제       저장소에
 데이터 추출                  적재
```

**특성**:
- 변환 후 적재 — 저장소에는 정제된 데이터만 저장
- 변환 로직이 파이프라인에 내장
- 소규모~중규모 데이터에 적합

### ELT (현대적)

**구조**:
```
[Extract] ──▶ [Load] ──▶ [Transform]
 소스에서     원본 그대로   저장소 내에서
 데이터 추출  적재         변환
```

**특성**:
- 원본 데이터 보존 — 나중에 다른 변환 가능
- 저장소의 컴퓨팅 파워 활용
- 대규모 데이터, 분석 목적에 적합

### Claude Code 적용

- **입력 데이터 보존 원칙**: 원본 녹음/발화 데이터는 변환 전 저장 (ELT 접근)
- **분석 결과는 파생 데이터**: 원본이 있으면 재생성 가능
- **파이프라인 레이어 설계 시**: Extract(캡처) → Load(DB 저장) → Transform(분석) 순서 고려

## 2. 이벤트 기반 아키텍처 (Event-Driven)

### 구조

```
[이벤트 발생] ──▶ [이벤트 버스/큐] ──▶ [이벤트 핸들러 A]
                                    ──▶ [이벤트 핸들러 B]
                                    ──▶ [이벤트 핸들러 C]
```

### 특성

- **비동기 처리**: 이벤트 발행자와 소비자가 분리
- **느슨한 결합**: 핸들러 추가/제거가 발행자에 영향 없음
- **확장성**: 핸들러를 독립적으로 스케일링

### 주요 패턴

| 패턴 | 설명 | 적용 |
|------|------|------|
| Pub/Sub | 발행자 → 구독자 | WebSocket 자막 브로드캐스트 |
| Event Sourcing | 모든 상태 변화를 이벤트로 기록 | 발화 이력 추적 |
| CQRS | 읽기/쓰기 모델 분리 | 실시간 표시 vs 분석 저장 |

### Claude Code 적용

- **실시간 스트리밍**: WebSocket → 이벤트 기반 자막 수신
- **발화 도착 이벤트**: `isFinal` 발화 → 분석 트리거
- **UI 업데이트**: Zustand store 변경 → React 리렌더링
- **적합한 레이어**: 입력층, 변환층 (실시간 데이터 흐름)

## 3. Lambda Architecture (람다 아키텍처)

### 구조

```
                ┌──▶ [Speed Layer] ──▶ 실시간 뷰
[데이터 입력] ──┤                        ↓
                └──▶ [Batch Layer] ──▶ [Serving Layer] ──▶ 통합 뷰
```

### 3개 레이어

| 레이어 | 역할 | 특성 |
|--------|------|------|
| **Speed Layer** | 실시간 근사 처리 | 낮은 지연시간, 근사값 허용 |
| **Batch Layer** | 정밀 배치 처리 | 높은 정확도, 처리 시간 허용 |
| **Serving Layer** | 결과 통합/제공 | 두 레이어 결과를 병합하여 제공 |

### Claude Code 적용 — **가장 일반적으로 사용되는 패턴**

- **Speed Layer (녹음 중)**:
  - 자막 스트리밍 (실시간 STT)
  - 경량 분류 (라벨링만)
  - UI 즉시 업데이트
- **Batch Layer (녹음 종료 후)**:
  - 전체 발화 대상 정밀 요약
  - 액션아이템 추출
  - 모순 탐지
  - 리서치 모드 보강
- **Serving Layer**:
  - 실시간 자막 + 배치 분석 결과를 통합 표시
  - 대시보드에 점진적 업데이트

### 실시간/배치 판별 기준

| 기준 | 실시간 (Speed) | 배치 (Batch) |
|------|----------------|-------------|
| 사용자 대면 여부 | O (즉각 피드백 필요) | X (백그라운드 처리 가능) |
| 정확도 요구 | 근사값 허용 | 높은 정확도 필수 |
| 데이터 범위 | 최근 N건 / 단일 이벤트 | 전체 데이터 |
| 처리 시간 | < 1초 | 수초~수분 허용 |
| 자원 소비 | 경량 | 고강도 허용 |

## 4. Circuit Breaker (회로 차단기)

### 구조

```
[호출자] ──▶ [Circuit Breaker] ──▶ [외부 서비스]
                   │
                   ├── Closed (정상): 요청 통과
                   ├── Open (차단): 즉시 실패 반환
                   └── Half-Open (시험): 일부 요청만 통과
```

### 3가지 상태

| 상태 | 동작 | 전환 조건 |
|------|------|----------|
| **Closed** | 모든 요청 통과 | 연속 N회 실패 → Open |
| **Open** | 즉시 실패 반환 (fallback) | 타이머 만료 → Half-Open |
| **Half-Open** | 일부 요청 시험 | 성공 → Closed / 실패 → Open |

### Claude Code 적용

- **외부 API 호출 보호**: Deepgram, Claude API, 리서치 API 등
- **Fallback 전략**:
  - STT 실패 → 오프라인 모드 안내
  - 분석 실패 → 기본 분류만 제공
  - 리서치 실패 → 보강 없이 결과 제공
- **에이전트 재작업 루프**: 2회 실패 → 에스컬레이션 (Circuit Breaker의 변형)

## 5. Saga Pattern (사가 패턴)

### 구조

```
[Step 1] ──▶ [Step 2] ──▶ [Step 3] ──▶ 완료
   │             │             │
   └──보상──◀──보상──◀──보상──◀── 실패 시 롤백
```

### 특성

- 분산 트랜잭션의 일관성 보장
- 각 단계에 대응하는 보상(취소) 작업 정의
- 실패 시 역순으로 보상 실행

### Claude Code 적용

- **멀티 에이전트 작업**: Worker A 완료 → Worker B 시작 → B 실패 시 A 결과 롤백 판단
- **파일 수정 롤백**: 에이전트가 여러 파일 수정 중 실패 → git으로 롤백
- **Phase 전환**: Phase N 완료 후 Phase N+1에서 문제 발견 → Phase N 재작업

## 6. Pipes and Filters (파이프-필터)

### 구조

```
[입력] ──pipe──▶ [Filter A] ──pipe──▶ [Filter B] ──pipe──▶ [Filter C] ──▶ [출력]
```

### 특성

- 각 Filter는 독립적이고 재사용 가능
- Filter 순서 변경/추가/제거 용이
- Unix 파이프라인 철학

### Claude Code 적용

- **파이프라인 레이어**: 각 레이어가 Filter 역할
- **분석 파이프라인**: 전처리 → STT → 분류 → 요약 → 보강
- **Filter 독립성**: 각 레이어가 정의된 입출력 타입만 준수하면 내부 구현 자유

## 패턴 선택 가이드

### 프로젝트 특성별 권장 패턴

| 특성 | 1순위 패턴 | 2순위 패턴 | 설명 |
|------|-----------|-----------|------|
| 실시간 데이터 처리 | Lambda | Event-Driven | 실시간 + 배치 분리 |
| 데이터 변환 파이프라인 | Pipes & Filters | ETL/ELT | 단계별 변환 |
| 외부 서비스 의존 | Circuit Breaker | Saga | 장애 격리 + 롤백 |
| 복잡한 워크플로 | Saga | Orchestrator | 트랜잭션 일관성 |
| 높은 확장성 | Event-Driven | Lambda | 비동기 + 느슨한 결합 |

### 조합 예시

- **음성 AI**: Lambda (실시간 STT + 배치 분석) + Pipes & Filters (분석 파이프라인) + Circuit Breaker (외부 API)
- **이커머스**: Event-Driven (주문 처리) + Saga (결제 워크플로) + Circuit Breaker (외부 결제 API)
- **교육**: ETL (학습 데이터) + Lambda (실시간 진도 + 배치 분석) + Pipes & Filters (콘텐츠 처리)
