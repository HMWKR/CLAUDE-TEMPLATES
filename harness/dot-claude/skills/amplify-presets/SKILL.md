# L1 인지 증폭 프리셋 (청사진 v2 — A/B 10라운드 실측 반영판)

## 태스크 유형별 N (모델 프로파일과 교차)

| 태스크 | 패턴 | Sonnet | Opus | Fable |
|--------|------|:---:|:---:|:---:|
| 설계·아키텍처 | 판정단(독립 시도→심사→승자) | 5 | 3 | 1–2 |
| 구현 | best-of-N + 실코드 테스트 게이트 | 3 | 2 | 1 |
| 리뷰·검수 | 렌즈 분산 × 반박자 | 4×3 | 4×2 | 3×2 |
| 디버그 | 경쟁 가설 | 5 | 4 | 3 |
| 모호성 해석 | 자기일관성 수렴 | 5 | 3 | 2 |

> **Fable 열 = 이력 참조**(2026-07-07 Fable 종료). 활성 모델 = **Opus(기본)·Sonnet**. Fable N값은 과거 A/B 실측 기록으로만 남긴다.

- **실측 주의(2026-07-03)**: 정확성 축에서 Opus는 N=1로도 Fable과 동률이었다 — Opus의 N은 평균이 아니라 **하방 변동성 보험**(간헐적 저점 1회/6 관측)이다. 중요도 낮은 작업은 Opus도 N=1 허용.

## 필수 규칙 (검수로 확정된 것)

1. **이종 판정단**: 반박자·선별자·채점자에 실행 모델과 **다른 모델 ≥1** 혼합(교차 채점) — 동종 다수결은 오류가 상관됨. 채점은 자기채점 금지.
2. **검증 주권 우선**: 기계 채점(실코드 러너·정답지)이 가능하면 판정단보다 우선한다 — 모델(과 채점자)의 주장을 신뢰 경로에서 제거. 채점 해석이 뒤집힐 수 있으니 원기록(agent transcript)을 증거로 보존.
3. **불확실성 보존**: 만장일치가 아니면 다수결로 뭉개지 말고 불확실 등급을 산출물에 보존.
4. **증폭 손익 백오프**: 앙상블 승자가 단일 표본 대비 개선 없음 2회 연속 → 해당 유형 N 하향. **승자 합성 기본 금지**(비양립 접근 병합 방지) — 승자 단독 채택, 합성은 판정자가 명시적 이득을 적시할 때만.
5. **발산 차단기**: 동일 실패 시그니처 3회 연속 → 중단 → ① 접근 강제 변경 ② 상위 모델 승격(sonnet→opus, 2026-07-07부터 fable 불가) ③ 인간 에스컬레이션. 토큰 무제한 선언(P2)도 이 차단기에 종속.
6. **카나리 캘리브레이션**(검수 루프): 심은 결함 탐지율 임계(2/3) 미달이면 "신규 발견 0"을 수렴으로 인정하지 않는다. 카나리 뱅크: `~/.claude/ab-bench/`(리뷰 7·hard 8·디자인 11·판단 5종).

## 실행 수단

네이티브 Workflow가 정본(교차 채점 패턴 실증 스크립트: `~/.claude/jobs/*/tmp/*.mjs` 참조 — critique-game·wide-sweep이 표준형). 프리셋 N은 L0 프로파일(`~/.claude/adapters/`)이 세션 모델에 맞게 이미 지시하므로, 워크플로 설계 시 이 표와 프로파일이 충돌하면 프로파일 우선.

## Fable-파리티 행동 규범 (서브에이전트 프롬프트에 병합 — 2026-07-05 실측 처방)

워크플로/팬아웃 설계 시 하위 모델(opus·sonnet) 작업자 프롬프트에 다음 3규범을 포함한다(A/B 20축 실측에서 유일하게 재현된 모델 차이 = 이 3가지 행동):
1. **출력 계약 엄수**: 요구된 형식만 출력(검증했더라도 부가 보고 금지) — 위반 실측 O79%/S33% vs F0%
2. **산출 전 자기검증**: 실행 가능하면 실행, 불가하면 손 추적 1회 — F가 2.9배 더 수행
3. **엣지 체크리스트**: 빈 입력·경계값·방향성(반올림/부호)·중복/소진 — 실측된 원샷 버그 전건이 엣지형
