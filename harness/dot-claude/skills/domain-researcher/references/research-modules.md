# 리서치 모듈 상세 템플릿

## 공통 출력 형식

모든 모듈의 마크다운 파일은 다음 헤더로 시작한다:

```markdown
# {모듈명}
> 조사일: {날짜} | 도메인: {도메인} | 검색 횟수: {N}회

## 요약 (3줄 이내)
{핵심 발견 사항}

## 신뢰도 범례
- [검색확인] = web_search로 직접 확인 + 출처 URL
- [내부지식] = 학습 데이터 기반, 검증 안 됨
- [추정] = 확인되지 않은 추론
- [상충] = 출처 간 정보 불일치
- [단일출처] = 1개 출처만 확인됨
```

## MODULE 1: 시장 분석 상세

### 검색 쿼리 생성 규칙

도메인 키워드를 다음 패턴에 삽입:
```
"{도메인} 시장 규모 2025"
"{도메인} 시장 전망 성장률"
"{도메인} 산업 현황 트렌드"
"한국 {도메인} 온라인 시장"
"{도메인} TAM SAM 스타트업"
```

### 출력 템플릿

```markdown
## 시장 규모
- 국내: {금액} ({연도} 기준) [검색확인] 출처: {URL}
- 글로벌: {금액} ({연도} 기준) [검색확인] 출처: {URL}
- 온라인 비중: {%} [추정]

## 성장률
- CAGR: {%} ({기간}) [검색확인] 출처: {URL}
- 성장 동인: {요인 1}, {요인 2}, {요인 3}

## 핵심 트렌드
1. {트렌드 1} — {설명} [검색확인]
2. {트렌드 2} — {설명} [내부지식]
3. {트렌드 3} — {설명} [검색확인]

## TAM/SAM/SOM
- TAM (전체): {금액} — {산출 근거}
- SAM (접근가능): {금액} — {산출 근거}
- SOM (초기 1년): {금액} — {산출 근거}
[추정] 위 수치는 시장 규모에서 역산한 추정치

## 계절성/주기성
{해당 사항 기술 또는 "뚜렷한 계절성 없음"}
```

## MODULE 2: 경쟁사 분석 상세

### 검색 쿼리 생성 규칙
```
"{도메인} {서비스유형} 서비스 비교 추천"
"{도메인} {서비스유형} 순위 TOP"
"{경쟁사명} 리뷰 장단점" (상위 5개 개별 검색)
"{도메인} 사용자 불만 개선점"
```

### 기능 비교 매트릭스 템플릿

```markdown
## 경쟁사 비교 매트릭스

| 기능 | 경쟁사A | 경쟁사B | 경쟁사C | 우리(계획) |
|------|---------|---------|---------|-----------|
| {핵심기능1} | O | O | X | O (차별화) |
| {핵심기능2} | X | △ | O | O |
| {핵심기능3} | X | X | X | O (독자) |
| 가격대 | {가격} | {가격} | {가격} | {계획} |
| 타겟 | {타겟} | {타겟} | {타겟} | {타겟} |

O = 제공, △ = 부분 제공, X = 미제공

## 미해결 문제 (차별화 기회)
1. {문제 1} — 기존 서비스들이 해결하지 못한 이유: {분석}
2. {문제 2} — ...
3. {문제 3} — ...
```

## MODULE 5: 규제/법률 상세

### 검색 쿼리 생성 규칙
```
"{도메인} 관련 법률 규제 한국 2025"
"{서비스유형} 사업자 신고 등록 요건"
"{도메인} 개인정보보호 데이터 수집 동의"
```

### 출력 템플릿

```markdown
## 필수 법적 요건

### 사업자 등록/신고
- {필요한 등록/신고 유형}: {근거 법률} [검색확인]

### 개인정보보호
- 수집 항목별 동의 요건: {설명}
- 보관 기간/파기 의무: {설명}
- 제3자 제공 동의: {설명}

### 도메인 특화 규제
| 규제 | 법률 | 핵심 내용 | 위반 시 |
|------|------|----------|---------|
| {규제명} | {법률명} | {내용} | {제재} |

### 면책 조항
이 문서는 참고용이며 법적 자문을 대체하지 않습니다.
실제 서비스 운영 전 변호사/법무법인의 검토를 권장합니다.
```

## 검색 실행 규칙

1. 검색 쿼리는 **1~6단어**로 짧게
2. 첫 검색 결과가 부족하면 키워드를 변형하여 재검색
3. 같은 쿼리를 반복하지 않음
4. web_fetch로 핵심 페이지의 상세 내용 확인
5. 연도가 중요한 정보는 반드시 최신 연도 포함
6. 영어 검색이 더 풍부한 분야(기술, 논문)는 영어 쿼리도 병행
