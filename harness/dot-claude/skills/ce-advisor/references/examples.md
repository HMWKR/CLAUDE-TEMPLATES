## 10. 고급 사용 패턴

### 연쇄 최적화 (Chained Optimization)

복잡한 작업에서 제안 3(Multi-Turn)을 선택한 후, 각 턴의 프롬프트를 다시 CE Advisor로 분석할 수 있다. 이 경우:

1. 첫 번째 CE Advisor 실행: 전체 작업을 Multi-Turn으로 분할
2. 각 턴별 프롬프트에 대해 CE Advisor를 재실행하여 개별 최적화
3. 최종 실행 시 각 턴이 독립적으로 최적화된 프롬프트로 동작

이 패턴은 대규모 코드 리팩토링, 다단계 분석 보고서, 복합 아키텍처 설계 등에 효과적이다.

### 컨텍스트 예산 기반 자동 추천

대화 길이에 따라 제안 유형을 자동으로 가중치 조절한다:

| 대화 상태 | 기본 추천 | 이유 |
|-----------|----------|------|
| 초반 (1-5턴) | 제안 2 (Context-Rich) | 컨텍스트 여유로 상세 정보 활용 가능 |
| 중반 (6-15턴) | 제안 1 (High-Signal) | 핵심만 전달하여 남은 컨텍스트 보전 |
| 후반 (16턴+) | 제안 3 (Multi-Turn) | 작업 분리로 컨텍스트 부담 분산 |

### 엣지 케이스 처리

**입력이 이미 최적인 경우**: 7축 분석에서 모두 충분하고 실패 모드 위험이 없으면, "현재 프롬프트가 이미 최적화되어 있습니다"라고 보고하고 미세 조정만 제안한다.

**입력이 극도로 부족한 경우**: Goal 축만 파악 가능하고 나머지가 모두 미흡하면, 제안 4(함께 구체화)를 강력 추천하고, 최소 3개의 구체적 질문을 준비한다.

**도메인 특화 프롬프트**: 법률, 의료, 금융 등 전문 도메인은 Constraints 축에 규제/준수 요건을 반드시 포함하도록 제안한다.

**기존 CLAUDE.md/Rules와의 충돌 감지**: 사용자의 프롬프트가 기존 글로벌 규칙이나 프로젝트 지침과 충돌하는 경우, Clash 실패 모드에서 구체적 충돌 지점을 명시하고 해소 방안을 제안한다.
