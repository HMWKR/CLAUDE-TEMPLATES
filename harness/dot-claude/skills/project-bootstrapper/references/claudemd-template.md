# CLAUDE.md 생성 규칙 (STAGE 2 상세)

## 필수 섹션

### 1. 프로젝트 개요
- 프로젝트명, 한 줄 설명
- 핵심 기능 3~5개 (번호 매겨서)
- 각 기능의 기술적 구현 방식 요약

### 2. 기술 스택
프로젝트 유형별 기본 추천:
- **프론트엔드**: Next.js 14+ (App Router) + Tailwind CSS + shadcn/ui + Zustand
- **백엔드**: Node.js(Express) 또는 Python(FastAPI) — AI 기능 있으면 Python 필수
- **DB**: PostgreSQL + Redis
- **인프라**: AWS (ECS Fargate + S3 + CloudFront + RDS)
- **CI/CD**: GitHub Actions
사용자가 다른 스택을 지정하면 그것을 따른다.

### 3. 코딩 컨벤션
- 공통: TypeScript/Python, 들여쓰기, 네이밍, 커밋 메시지, 에러 처리
- 프론트: 컴포넌트 패턴, 폴더 구조, API 호출, 접근성
- 백엔드: API 응답 형식, 인증, 로깅, rate limiting, 입력 검증
- 도메인별 추가 (AI서비스: GPU 관리, 모델 로딩 등)

### 4. 아키텍처 원칙
프로젝트에서 가장 중요한 기술적 원칙 5~6개.
예: 마이크로서비스 분리, 비동기 처리, 캐시 우선, 실패 우아하게 등.

### 5. 에이전트 팀 구조
- 부서별 리드 + 인원수 테이블
- 에이전트 호출 방법 (코드 예시)
- 의사결정 우선순위 위계

### 6. 금지사항
프로젝트별 핵심 리스크에서 도출. 최소 6개.
예: 동기 API로 heavy 작업 금지, 원본 데이터 영구 저장 금지 등.

### 7. 참조 경로
도메인 날리지, 워크플로우 파일 위치.
