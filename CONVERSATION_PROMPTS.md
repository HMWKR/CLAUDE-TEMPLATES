## 1. 기본 작업 모드

### Ultrathink 모드 (깊은 분석)

복잡한 작업이나 중요한 결정이 필요할 때 사용합니다.

```
ultrathink 모드로 [작업 내용]에 대해 분석해줘.
```

**예시:**
```
ultrathink 모드로 현재 인증 시스템의 보안 취약점을 분석해줘.
```

### 계획 모드 (단계별 접근)

큰 작업을 시작하기 전 계획을 세울 때 사용합니다.

```
[작업 내용]에 대한 계획을 세워줘. 구현은 하지 말고 계획만.
```

**예시:**
```
사용자 인증 시스템 구현에 대한 계획을 세워줘. 구현은 하지 말고 계획만.
```

### 검수 모드 (품질 검증)

작업 완료 후 전체 검수가 필요할 때 사용합니다.

```
[작업 범위]에 대해 시니어 개발자 관점에서 전체 검수해줘.
```

## 2. 기능 구현 프롬프트

### 기본 패턴

```
[기능명]을 구현해줘.

## 요구사항
- [요구사항 1]
- [요구사항 2]

## 제약사항
- [제약사항 1]
```

### 상세 패턴 (복잡한 기능)

```
## 역할
너는 10년 경력의 [관련 분야] 시니어 개발자야.

## 목표
[기능명]을 구현해줘.

## 기술 요구사항
- [상세 스펙 1]
- [상세 스펙 2]

## 금지 사항
- [하지 말아야 할 것 1]
- [하지 말아야 할 것 2]

## 테스트
- 단위 테스트도 함께 작성해줘
```

**예시:**
```
## 역할
너는 10년 경력의 React 시니어 개발자야.

## 목표
다크모드 토글 기능을 구현해줘.

## 기술 요구사항
- Context API로 전역 상태 관리
- localStorage에 테마 설정 저장
- 시스템 설정 자동 감지 (prefers-color-scheme)
- Tailwind CSS dark: 접두사 활용

## 금지 사항
- 인라인 스타일 사용 금지
- any 타입 사용 금지

## 테스트
- 단위 테스트도 함께 작성해줘
```

## 11. 프롬프트 품질 검수 모드

### 5계층 48점 평가 시스템

프롬프트 품질을 객관적으로 평가하는 시스템입니다.

```
┌─────────────────────────────────────────────────────────────┐
│  계층 5: 할루시네이션 검증 (12점) - 게이트키퍼              │
│  계층 4: 효과 검증 (4점)                                    │
│  계층 3: 맥락 최적화 (6점)                                  │
│  계층 2: 엔지니어링 기법 (8점)                              │
│  계층 1: 구조적 완성도 (18점) - 필수 100%                  │
└─────────────────────────────────────────────────────────────┘
```

### 기본 패턴 - 프롬프트 평가 요청

```
내가 작성한 프롬프트를 5계층 48점 시스템으로 평가해줘.

## 평가할 프롬프트
[프롬프트 내용]

## 요청
1. 계층별 점수 부여
2. 부족한 부분 지적
3. 개선된 프롬프트 제안
```

### 상세 패턴 - 할루시네이션 검증 요청

```
이 응답에 대해 할루시네이션 검증을 수행해줘.

## 검증 대상
[검증할 내용]

## 검증 항목
1. 코드/파일 정확성
   - 언급된 파일이 실제 존재하는가?
   - 함수/변수명이 정확한가?
   - 라인 번호가 맞는가?

2. 기술 정보 정확성
   - API 사용법이 정확한가?
   - 버전 정보가 맞는가?

3. 논리/추론 정확성
   - 인과관계가 타당한가?
   - 모순이 없는가?

## 출력 형식
| 항목 | 검증 결과 | 마커 |
|------|----------|------|
| [항목] | [결과] | [검증됨/추정/미확인] |
```

### 프롬프트 최적화 요청

```
다음 프롬프트를 최적화해줘.

## 원본 프롬프트
[원본 내용]

## 최적화 방향
- 구조적 완성도 향상 (18점 만점 목표)
- 엔지니어링 기법 적용 (CoT, Role, Few-shot 등)
- 명확한 제약 조건 추가

## 산출물
1. 최적화된 프롬프트
2. 개선 포인트 설명
3. Before/After 점수 비교
```

### 등급별 기준

| 등급 | 점수 | 기준 |
|:----:|:----:|------|
| **S** | 46-48 | 최고 품질 - 모든 계층 우수 |
| **A+** | 42-45 | 우수 - 계층 1 만점 + 대부분 충족 |
| **A** | 38-41 | 양호 - 구조적 완성도 높음 |
| **B+** | 34-37 | 보통 - 기본 요소 충족 |
| **B** | 30-33 | 개선 필요 - 일부 누락 |
| **F** | <30 | 재작성 필요 |

### 검증 마커 사용 요청

```
다음 설명에 검증 마커를 추가해줘.

## 검증 마커 종류
- [검증됨]: Glob/Grep/Read로 직접 확인
- [문서확인]: 공식 문서에서 확인
- [추정]: 패턴 기반 추측
- [미확인]: 추후 검증 필요
- [가정]: 전제 조건 명시

## 대상
[마커를 추가할 내용]
```

## 12. 프롬프트 저널 작성 모드

16섹션 커밋과 병행하여 **읽고 배우기 좋은** 프롬프트 저널을 작성합니다.

### 기본 패턴 - 저널 작성 요청

```
오늘 작업에 대한 프롬프트 저널을 작성해줘.
PROMPT_JOURNAL_TEMPLATE.md 형식을 따라서.
```

### 상세 패턴 - 특정 작업 저널화

```
방금 완료한 [기능/작업]에 대한 프롬프트 저널을 작성해줘.

## 포함 내용
- 원본 프롬프트 (내가 요청한 그대로)
- 사고 여정 (처음 생각 → 전환점 → 최종 통찰)
- 최적화된 프롬프트 (개선된 버전)
- 핵심 학습 (배운 점)
- 결과 분석 (성공/실패, 반복 횟수)

## 형식
- YAML frontmatter 포함
- 파일명: .prompts/[날짜]-[주제].md
```

### 회고 패턴 - 과거 저널 기반 학습

```
.prompts/ 폴더의 저널들을 분석해서
내가 자주 하는 실수나 개선할 패턴을 찾아줘.
```

### 저널 품질 검수

```
이 프롬프트 저널을 검수해줘.

## 검수 항목
- YAML frontmatter 완성도
- 사고 여정의 명확성
```
